# Oks-Coding-System 使用指南

AI Agent 开发工具系统，包括工作流命令、功能模板和编写规范。

---

## 🆕 创建新项目（入口命令）

**`/oks-new` 是整个工作流的入口命令**，采用交互式引导：

### 使用方式

```bash
# 方式一：直接描述需求
/oks-new 我需要开发一个财务管理模块，包含记账、报表功能

# 方式二：交互式引导（不带参数）
/oks-new
```

### 交互流程示例

```
用户: /oks-new 我需要开发购物车功能，支持商品管理、价格计算、优惠券

AI: 【提炼结果】
    项目名称: shopping-cart (购物车)

    功能模块:
    - M-001: 商品管理 (P1) - 添加/删除/修改商品
    - M-002: 价格计算 (P1) - 总价/小计计算
    - M-003: 优惠券 (P2) - 优惠券应用

    目标用户: 购物用户

    【是否确认创建？】

用户: 确认

AI: ✅ 项目已创建！
    📄 产品愿景: features/shopping-cart/vision.md
    📊 实现进度: features/shopping-cart/implementation.md
```

### 生成文档

| 文档                | 视角 | 内容                     |
| ------------------- | ---- | ------------------------ |
| `vision.md`         | 业务 | 是什么、为什么、业务场景 |
| `implementation.md` | 进度 | 完成情况、下一步         |

---

## 🔒 阶段防护机制

每个阶段命令都会自动验证：

1. **前置阶段是否完成** - 确保工作流顺序执行

**阶段依赖**:

```
/oks-new → 阶段一 (user-story) → 阶段二 (bdd) → 阶段三 (tdd) → 阶段四 (implementation) → 阶段五 (optimization)
```

| 阶段                    | 前置条件                  |
| ----------------------- | ------------------------- |
| 阶段一 (user-story)     | 项目已初始化（自动创建）  |
| 阶段二 (bdd)            | 用户故事文档              |
| 阶段三 (tdd)            | 用户故事 + BDD 场景(可选) |
| 阶段四 (implementation) | TDD 模块                  |
| 阶段五 (optimization)   | 实现代码                  |

## 🚀 快速开始

### 1. 创建新项目

```bash
/oks-new 我需要开发财务管理功能，包含记账和报表
```

### 2. 确认项目信息

AI 会展示提炼的项目名称和功能模块，确认后创建项目。

### 3. 开始开发

```bash
# 完整工作流（推荐）
/oks-workflow 财务管理

# 或单独执行某个阶段
/oks-stage-1-user-story 财务管理
```

### 4. 查看进度

```bash
/oks-status 财务管理
```

## 🔄 完整工作流程

```
┌─────────────┐
│  /oks-new   │  ← 入口：交互式需求收集
│  vision.md  │     AI 提炼项目名称和功能模块
└──────┬──────┘     用户确认后创建项目
       │
       ▼
┌─────────────┐
│  阶段一     │  用户故事 (基于 vision.md)
│  design.md  │  技术设计
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  阶段二     │  BDD 场景 (Gherkin)
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  阶段三     │  TDD 循环 (Red-Green-Refactor)
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  阶段四     │  代码实现
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  阶段五     │  代码优化
└─────────────┘
```

**总耗时**: 2-4 小时 | **测试覆盖率**: > 80%

## 📊 技术栈

本工具系统采用以下现代化开发实践：

- ✅ **TDD** (测试驱动开发) - 红-绿-重构循环
- ✅ **BDD** (行为驱动开发) - Gherkin 语法
- ✅ **DDD** (领域驱动设计) - 实体、值对象、聚合
- ✅ **Clean Architecture** - 分层架构
- ✅ **INVEST 原则** - 用户故事验证
- ✅ **ADR** (架构决策记录) - 决策文档
