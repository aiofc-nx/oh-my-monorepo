# tp_nestjs_mvc 模板使用指南

本目录包含用于 NestJS MVC 架构的开发模板文件。

---

## 📁 模板文件列表

| 文件                | 用途            | 使用时机                   |
| ------------------- | --------------- | -------------------------- |
| `vision.md`         | 产品愿景文档    | 阶段一：愿景               |
| `user-story.md`     | 用户故事文档    | 阶段二：用户故事           |
| `design.md`         | 技术设计文档    | 阶段三：技术设计           |
| `bdd-scenarios.md`  | BDD 场景定义    | 阶段四：BDD 场景（可跳过） |
| `implementation.md` | 实现进度跟踪    | 阶段五至七：开发过程       |
| `decisions.md`      | 架构决策记录    | 全程（需要记录重要决策时） |
| `AGENTS.md`         | AI 助手行为指南 | 全程（AI 助手参考）        |

---

## 🚀 快速开始

### 使用 OKS 命令

```bash
# 完整工作流
/oks-workflow {功能名称}

# 或分阶段执行
/oks-vision {项目名}
/oks-user-story {功能名称}
/oks-design {功能名称}
/oks-bdd {功能名称}
/oks-tdd {功能名称}
/oks-implementation {功能名称}
/oks-optimization {功能名称}
```

## 📝 占位符说明

### 核心占位符

| 占位符        | 说明                       | 示例值        |
| ------------- | -------------------------- | ------------- |
| `{功能名称}`  | 功能的中文名称             | `用户登录`    |
| `{feature}`   | 功能的英文标识（用于路径） | `user-login`  |
| `{project}`   | 项目名称                   | `user-center` |
| `{日期}`      | 日期                       | `2026-03-12`  |
| `{module}`    | 模块名称                   | `user`        |
| `{component}` | 组件名称                   | `LoginForm`   |

### 场景相关

| 占位符       | 说明           | 示例值                 |
| ------------ | -------------- | ---------------------- |
| `{场景名称}` | BDD 场景的名称 | `用户使用正确凭据登录` |
| `{场景数量}` | 场景的数量     | `5`                    |
| `{前置条件}` | 场景的前置条件 | `用户在登录页面`       |
| `{执行动作}` | 用户的操作     | `输入邮箱和密码`       |
| `{期望结果}` | 预期的结果     | `成功登录并跳转`       |

### 数值相关

| 占位符       | 说明                   | 示例值 |
| ------------ | ---------------------- | ------ |
| `{覆盖率}`   | 测试覆盖率             | `85`   |
| `{场景数量}` | 场景数量               | `12`   |
| `{N}`        | 数值（在特定上下文中） | `100`  |

### 描述相关

| 占位符       | 说明       | 使用场景   |
| ------------ | ---------- | ---------- |
| `{简要描述}` | 一句话描述 | 功能概述   |
| `{说明}`     | 通用说明   | 各种场景   |
| `{描述}`     | 详细描述   | 背景、需求 |
| `{原因}`     | 原因说明   | 为什么做   |
| `{如何处理}` | 处理方式   | 边界情况   |

## 🔄 工作流程（7 阶段）

### 阶段一：愿景文档

**命令**: `/oks-vision`

**产出**: `docs/specfiy/vision.md`

**内容**: 产品愿景、目标用户、核心功能

### 阶段二：用户故事

**命令**: `/oks-user-story {功能名称}`

**产出**: `docs/specfiy/user-story.md`

**内容**: INVEST 用户故事、功能需求、验收标准

### 阶段三：技术设计

**命令**: `/oks-design {功能名称}`

**产出**: `docs/specfiy/design.md`

**内容**: 数据库设计、API 设计、组件设计、边界情况

### 阶段四：BDD 场景

**命令**: `/oks-bdd {功能名称}`（可跳过）

**产出**: `features/{feature}.feature`

**内容**: Gherkin 场景、步骤定义

### 阶段五：TDD 开发

**命令**: `/oks-tdd {功能名称}`

**产出**: `src/modules/{module}/entities/`

**内容**: 领域实体、值对象、单元测试

### 阶段六：服务实现

**命令**: `/oks-implementation {功能名称}`

**产出**: `src/modules/{module}/services/`, `controllers/`, `repositories/`

**内容**: 服务层、控制器、数据访问层

### 阶段七：代码优化

**命令**: `/oks-optimization {功能名称}`（可跳过）

**内容**: 性能优化、代码质量、安全加固

## 💡 使用建议

### DO ✅

- 在开始开发前先填写设计文档
- 使用 TDD 方式开发
- 及时更新 `implementation.md` 进度
- 记录重要技术决策
- 使用命令中的常用提示词

### DON'T ❌

- 不要跳过设计直接开发
- 不要忽略边界情况
- 不要忘记更新进度文档
- 不要删除模板中的占位符（除非确定不需要）
