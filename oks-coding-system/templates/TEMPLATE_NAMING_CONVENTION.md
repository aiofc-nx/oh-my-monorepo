# 模板占位符命名规范

本文档定义 `oks-coding-system/templates/` 目录下所有模板文件的占位符命名规范。

---

## 1. 占位符语法

- **使用 `{}` 包裹**：`{功能名称}`、`{日期}`
- **不使用 `[]`**：~~`[功能名称]`~~（避免与命令行"可选参数"混淆）
- **不使用 `{{}}`**：~~`{{功能名称}}`~~（避免与模板引擎混淆）

---

## 2. 命名约定

### 2.1 通用原则

- ✅ **清晰明确**：一眼就能看出要填什么
- ✅ **保持一致**：相同概念使用相同名称
- ✅ **简洁适中**：不要太长也不要太短

### 2.2 推荐格式

| 类型     | 格式             | 示例                       |
| -------- | ---------------- | -------------------------- |
| 中文项目 | `{中文名称}`     | `{功能名称}`、`{日期}`     |
| 英文项目 | `{ENGLISH_NAME}` | `{FEATURE_NAME}`、`{DATE}` |
| 混合项目 | 优先中文         | `{功能名称}`、`{Date}`     |

## 3. 核心占位符列表

### 3.1 项目信息

| 占位符           | 说明            | 示例值       |

| 占位符        | 说明                 | 示例值        |

| 占位符       | 说明       | 使用场景   |

### 3.4 数量和数值

## 4. 常见模式

```

```

## 5. 避免的用法

### ❌ 不推荐的占位符

```markdown
{1} # 太抽象，看不出含义
{xxx} # 无意义的占位符
{TODO} # 应该用具体的待办事项
{FIXME} # 应该用具体的问题描述
{功能名} # 不一致，应该用 {功能名称}
{feature name} # 不一致，应该用 {功能名称} 或 {FEATURE_NAME}
```

### ✅ 推荐的替代方案

```markdown
{步骤编号} # 用有意义的名称
{待填写} # 明确的占位符
{具体描述} # 具体的描述
{功能名称} # 统一使用完整名称
{FEATURE_NAME} # 统一使用大写和下划线
```

## 7. 检查和验证

### 验证占位符格式

```bash
# 查找所有占位符
grep -oh '{[^}]*}' templates/**/*.md

# 查找不一致的占位符
grep -oh '{功能名[^}]*}' templates/**/*.md
```

### 常见问题

1. **Q: 一个项目中可以混用中文和英文占位符吗？**
   A: 可以，但建议统一。推荐优先使用中文。

2. **Q: 占位符应该用单数还是复数？**
   A: 根据上下文决定。例如：`{用户角色}` vs `{用户角色列表}`

3. **Q: 如何处理动态数量的列表？**
   A: 使用编号：`{项目1}`、`{项目2}`、`{项目N}`

## 8. 更新模板时的检查清单

- [ ] 所有占位符使用 `{}` 格式
- [ ] 没有使用 `[]` 或 `{{}}`
- [ ] 占位符命名一致（例如都使用 `{功能名称}` 而不是混用 `{功能名}`)
- [ ] 占位符清晰明确
- [ ] 没有无意义的占位符（如 `{xxx}`）
- [ ] 遵循本项目选择的命名规范（中文/英文/混合）
