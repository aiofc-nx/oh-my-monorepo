# 用户登录设计

> 单一事实来源 (Source of Truth) — 所有实现以此文档为准

---

## 概述

实现用户登录功能，允许注册用户使用邮箱和密码登录系统，访问个人数据和功能。支持"记住我"功能（7 天免登录）和账户安全保护机制（连续失败 5 次后锁定 15 分钟）。

---

## 问题陈述

**当前问题**:
用户无法登录系统，无法访问个人数据和功能。

**为什么需要解决**:
登录是用户访问系统的核心入口，没有登录功能用户无法使用任何需要身份验证的功能。

**影响范围**:
所有需要身份验证的功能，包括用户资料、订单、设置等。

---

## 用户故事（INVEST 原则）

### 主要用户故事

```gherkin
作为注册用户
我想要使用邮箱和密码登录系统
以便于访问我的个人数据和功能
```

**优先级**: P1

**优先级原因**: 登录是用户访问系统的核心入口，没有登录用户无法使用任何功能。

**独立测试**: 可以通过输入有效邮箱和密码完成登录测试，验证用户能够成功进入系统首页。

### 功能需求

- **FR-001**: 系统必须允许用户使用邮箱和密码登录
- **FR-002**: 系统必须验证邮箱格式和密码正确性
- **FR-003**: 系统必须在登录失败时显示明确的错误信息
- **FR-004**: 系统必须支持"记住我"功能（7 天免登录）
- **FR-005**: 系统必须在连续失败 5 次后锁定账户 15 分钟

### 待澄清项

- **FR-006**: 系统必须支持第三方登录 - [NEEDS CLARIFICATION: 未明确支持哪些第三方平台（Google/GitHub/WeChat?）]

## 技术设计

### 数据库变更

**修改表**: `users`

```sql
-- 添加登录失败计数字段
ALTER TABLE users ADD COLUMN login_attempts INTEGER DEFAULT 0;
ALTER TABLE users ADD COLUMN is_locked BOOLEAN DEFAULT FALSE;
ALTER TABLE users ADD COLUMN locked_until TIMESTAMP;

-- 索引
CREATE INDEX idx_users_email ON users(email);
```

**迁移文件**: `add_login_security_fields`

**影响**: 现有用户数据不受影响，新字段使用默认值。

### API 变更

**新增接口**:

```typescript
// POST /api/auth/login
Request: {
  email: string;        // 必填，邮箱格式
  password: string;     // 必填，至少 8 个字符
  rememberMe?: boolean; // 可选，默认 false
}

Response (Success - 200): {
  token: string;        // JWT Token
  expiresIn: number;    // 过期时间（毫秒）
  user: {
    id: string;
    email: string;
    name: string;
  }
}

Response (Error - 401): {
  error: string;        // "邮箱或密码错误" 或 "账户已锁定"
  code: "INVALID_CREDENTIALS" | "ACCOUNT_LOCKED"
}
```

**向后兼容性**: 新接口，向后兼容。

### UI 变更

**登录页面**:

- **组件**: `LoginForm`
- **字段**:
  - 邮箱输入框（必填）
  - 密码输入框（必填）
  - "记住我"复选框
  - 登录按钮

**用户流程**:

1. 用户访问登录页面
2. 输入邮箱和密码
3. （可选）勾选"记住我"
4. 点击登录按钮
5. 系统验证并返回结果（成功/失败）

## 边界情况

### 输入边界

- **邮箱为空/null**: 显示错误"邮箱不能为空"
- **密码为空/null**: 显示错误"密码不能为空"
- **邮箱格式不正确**: 显示错误"邮箱格式不正确"
- **密码超长（> 1000 字符）**: 显示错误"密码长度不能超过 1000 个字符"

### 状态边界

- **用户不存在**: 显示错误"邮箱或密码错误"（不暴露用户是否存在）
- **密码错误**: 显示错误"邮箱或密码错误"，增加失败计数
- **账户被锁定**: 显示错误"账户已锁定，请 15 分钟后再试"
- **登录失败 5 次**: 锁定账户 15 分钟

### 权限边界

- **未激活账户**: 允许登录（激活不是登录的前置条件）
- **被禁用账户**: 显示错误"账户已被禁用"（如果实现了禁用功能）

### 系统边界

- **数据库连接失败**: 显示错误"系统错误，请稍后重试"
- **JWT 密钥缺失**: 返回 500 错误，记录日志
- **Token 生成失败**: 返回 500 错误，记录日志

## 成功标准

### 功能成功标准

- **SC-001**: 用户可以在 30 秒内完成登录流程
- **SC-002**: 登录 API 响应时间 < 200ms
- **SC-003**: 90% 的用户在首次尝试时成功登录
- **SC-004**: 登录相关安全事件 100% 被记录

### 质量成功标准

- **代码覆盖率**: > 80%
- **代码复杂度**: < 10
- **性能**: API 响应时间 < 200ms
- **安全**: 密码使用 bcrypt 加密，无明文存储

## 范围外

以下内容明确**不包含**在本功能中：

- **第三方登录**: Google/GitHub/WeChat 登录（后续功能）
- **多因素认证（MFA）**: 邮箱验证码/手机验证码（后续功能）
- **密码重置**: 忘记密码功能（独立功能）
- **用户注册**: 新用户注册（独立功能）
- **用户登出**: 登出功能（独立功能）
- **改期原因配置**: 与登录无关（超出范围）

## 数据流

```
1. 用户输入邮箱和密码
   ↓
2. 控制器验证输入格式
   ↓
3. 服务层查询用户（通过邮箱）
   ↓
4. 检查账户锁定状态
   ↓
5. 验证密码（bcrypt.compare）
   ↓
6. 更新登录失败计数（如果失败）
   ↓
7. 生成 JWT Token（如果成功）
   ↓
8. 重置登录失败计数（如果成功）
   ↓
9. 返回 Token 和用户信息
```

**关键数据**:

- `email`: 用户邮箱，用于查找用户
- `password`: 用户密码，用于验证身份
- `rememberMe`: 是否记住登录，影响 Token 有效期
- `loginAttempts`: 登录失败计数，用于安全保护
- `isLocked`: 账户锁定状态

## 风险和假设

### 风险

| 风险                   | 影响 | 概率 | 缓解措施                  |
| ---------------------- | ---- | ---- | ------------------------- |
| 暴力破解密码           | 高   | 中   | 登录失败计数 + 账户锁定   |
| 密码泄露               | 高   | 低   | bcrypt 加密，定期更换密钥 |
| JWT Token 被盗用       | 中   | 低   | 短期 Token，HTTPS 传输    |
| 数据库性能问题影响登录 | 中   | 低   | 添加索引，优化查询        |

### 假设

- 用户已通过注册功能创建账户
- 密码在注册时已使用 bcrypt 加密存储
- 系统已配置 HTTPS
- JWT_SECRET 环境变量已设置
